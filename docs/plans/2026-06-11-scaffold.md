## Design decisions (one at a time)

**D1 — Scaffolding method.** Use `npm create vite@latest mos-app -- --template react-ts`, then prune/replace the generated boilerplate (delete `App.css`, the Vite/React logo SVGs, the default `App.tsx`/`index.css` bodies) rather than hand-rolling `package.json`. Rationale: the template gives a correct `tsconfig` project-references split and a known-good `main.tsx` mount; pruning is faster and less error-prone than authoring from zero. The template's React/Vite versions are then explicitly re-pinned (T-002) so we control majors, not `create vite`'s drift.

**D2 — Inter font: self-host via `@fontsource-variable/inter`.** Chosen over (a) Google Fonts CDN and (b) system-fallback-only.
- vs CDN: no third-party request on a path-mounted internal ops app behind `ops.gordi.id`; deterministic offline/dev rendering; no FOUT from a cross-origin stylesheet. DESIGN.md makes Inter the single UI font ("Inter everywhere") — it must be present, not best-effort.
- vs system-only: the fallback stack (`system-ui, -apple-system, "Segoe UI", sans-serif`) ships as the CSS fallback regardless, but relying on it for the *primary* face would let the design drift per-OS. Variable font = one file, all weights (400/500/600/700 all used in DESIGN.md) without four static imports. One `import '@fontsource-variable/inter'` in `main.tsx`; `font-family` keeps the full fallback chain so a font-load failure degrades gracefully.

**D3 — Router shape.** `createBrowserRouter` with `basename="/mos"` (router basename) + `vite.config.ts` `base: '/mos/'` (asset base). One route `/` → `<Home/>` placeholder. `basename` is the load-bearing seam: it lets every in-app `<Link to="/tasks">` resolve to `/mos/tasks` without per-link prefixing, matching OD-P0-5's path mount. This is the same router that P1-4 grows the real shell into — no rewrite.

**D4 — Test layering for the scaffold.** Vitest + RTL + jsdom owns the unit smoke (render `App`, assert `heading` role) — fast, no browser. Playwright owns ONE real cross-stack smoke (dev server actually serves `/mos/`, real navigation, heading visible). This is the test-pyramid shape the project mandates, established at scaffold time so later issues slot in. No `AC-###` to tag (infra), but each test asserts a real user-visible fact, not a tautology.

**D5 — No ADR.** Nothing here is architecturally irreversible or cross-cutting beyond what `DESIGN.md` + `docs/decisions.md` already ratify (token pipeline, `/mos` mount, design system). Version pins and font choice are documented in this plan and reversible. Per CLAUDE.md ("ADRs: only for architectural / irreversible / cross-cutting decisions"), no `docs/adr/NNNN-*.md` is created.
